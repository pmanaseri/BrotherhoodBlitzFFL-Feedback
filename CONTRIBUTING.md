# Reporting Guidelines

Thanks for helping improve Brotherhood Blitz FFL.

## Good Reports

A good report says:

- Where the issue happened.
- What you were trying to do.
- What went wrong.
- What should have happened instead.
- Whether it happens every time or only sometimes.

Screenshots help a lot for UI, visual, roster, scoring, and standings issues.

## Public Tracker Reminder

This is a public feedback area. Do not include passwords, login links, private account details, private league financial details, API keys, tokens, or anything sensitive.

## Report Types

- Use Bug Report when something is broken.
- Use Feature Request when the site does not do something yet.
- Use Scoring / Stats Issue for scores, standings, matchups, projections, records, or player stat problems.
- Use Roster / Owner / Team Issue for owner identity, team names, team pages, roster display, or profile/team mapping.
- Use Visual / UI Issue for layout, styling, mobile layout, readability, or confusing controls.
- Use Speed / Performance Issue for slow loading, timeouts, or pages that feel stuck.
- Use General / Missing when you are not sure where the report belongs.

## Triage Flow

Maintainers will usually:

1. Confirm the issue is clear enough to reproduce or evaluate.
2. Apply the right `component:*`, `impact:*`, and `priority:*` labels.
3. Move the issue through the project board.
4. Ask for more info when the report is missing key details.
5. Close the issue when fixed, duplicated, not planned, or no longer actionable.
